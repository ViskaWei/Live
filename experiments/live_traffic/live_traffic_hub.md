## 2) 口径冻结（唯一权威）

> 详细 KPI 体系见 [node_kpi.md](./node_kpi.md)

### 2.1 场景与目标

| 项目 | 规格 |
|---|---|
| 场景 | TikTok Live US - 主播供给增长 |
| 问题本质 | 4类冷启动 + 双边市场 + 探索-利用(E&E) |
| 增长双线 | 拉新(Activation) + 成长(Growth/Survival) |
| 优化目标 | 用户侧 + 主播侧 + 平台侧 LTV |
| 时间窗口 | 启动期(Session Cold Start) 10-30 分钟 |

### 2.2 KPI 速览

| 层级 | KPI | 说明 |
|------|-----|------|
| 主KPI | HP Supply 增长 | 7天>=2场 & >=60min 的高潜主播人数 |
| 主KPI | HP 30天生存率 | 新高潜主播30天后仍有效开播的比例 |
| 护栏 | 用户负反馈率 | 不升（尤其新用户） |
| 护栏 | 生态多样性 | 不恶化 |

### 2.3 高潜主播口径

> **业务定义**：高潜主播 = 未来30天能稳定开播 + 不伤用户体验 + 有增长信号

| 维度 | 条件 | 示意阈值 |
|------|------|---------|
| 生存 | 未来30天内>=N场有效直播 | N=4 |
| 体验 | 负反馈率<=R，无高风险违规 | R=p50 |
| 成长 | 粉丝净增>=F 或 回访>=V 或 互动>=I | 满足其一 |

> 规则：任何口径变更必须写入 §8 变更日志。

### 术语表

| 术语 | 英文 | 定义 |
|---|---|---|
| 双边市场 | Two-sided Market | 同时服务用户（观众）与主播（内容生产者）的平台 |
| 新主播冷启动 | Creator Cold Start | 主播刚开始没历史数据 |
| 单场冷启动 | Session Cold Start | 每场开播前5-20分钟信息稀薄 |
| 新观众冷启动 | User Cold Start | 新用户意图识别困难 |
| 双边冷启动 | Marketplace Cold Start | 同时养供给+保需求 |
| Room State | - | 直播间当前状态（观众/互动/氛围） |
| 探索-利用 | Exploration-Exploitation | 新内容需探索，但会损失即时收益 |
| 高潜主播 | High Potential | 满足"生存+体验+成长"的主播 |
| 三段式机制 | Seed-Validate-Scale | 播种→验证→加码 |
| 成长信号 | Growth Signal | 非金钱激励：粉丝、互动、里程碑 |

### 背景
- US直播市场特性：打赏收入有限，主播动机需靠成长/认同/粉丝反馈支撑
- 核心约束：不能为高付费场景牺牲低付费/纯观看用户体验
- 核心目标：在约束下实现用户体验 + 主播留存的双赢
