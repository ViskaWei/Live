## 1.2 Gate定义

### Gate-1: 启动期三段式机制可行性

| 项 | 内容 |
|----|------|
| 验证 | 三段式（播种-验证-加码）能否有效保障用户体验同时给主播机会 |
| MVP | MVP-0.1, MVP-0.2 |
| 若通过 | 留存稳定 + 负反馈可控 → 上线三段式机制 |
| 若失败 | 体验受损严重 → 调整门槛或回滚 |
| 状态 | ⏳ |

### Gate-2: 高潜主播识别模型效度

| 项 | 内容 |
|----|------|
| 验证 | 潜力评分卡能否有效预测30/60天主播LTV或生存概率 |
| MVP | MVP-1.1, MVP-1.2, MVP-1.3 |
| 若AUC>0.7 | 上线分层探索预算 → S/A层配置更高探索预算 |
| 若AUC<0.6 | 需增加特征或调整标签定义 |
| 状态 | ⏳ |

### Gate-3: 用户意图分层排序有效性

| 项 | 内容 |
|----|------|
| 验证 | 差异化排序权重是否提升用户体验 + 转化效率 |
| MVP | MVP-2.1, MVP-2.2 |
| 若Δ体验>5% | 上线用户分层策略 |
| 若无显著差异 | 简化为粗分层或放弃 |
| 状态 | ⏳ |

### Gate-4: 成长信号产品化对留存的因果效应

| 项 | 内容 |
|----|------|
| 验证 | 成长信号（仪表盘/里程碑/可解释流量）能否提升主播开播留存 |
| MVP | MVP-3.1, MVP-3.2 |
| 若Δ留存>10% | 优先级提升，全量产品化 |
| 若Δ留存<5% | 仅保留基础功能 |
| 状态 | ⏳ |

# 3. 🔧 MVP规格

## Phase 0: 基础建设

### MVP-0.1: 指标体系定义

| 项 | 配置 |
|----|------|
| 目标 | 定义多目标优化的完整指标体系 |
| 用户侧指标 | 观看时长、有效互动（评论/关注/停留）、次日/7日回访、负反馈率 |
| 主播侧指标 | 开播留存(7/30日)、开播频次、内容稳定性、粉丝增长、互动质量、收入 |
| 平台侧指标 | 长期LTV（用户+主播）、生态多样性 |
| 验收 | 完成指标字典文档 |

### MVP-0.2: 启动期门槛数据分析

| 项 | 配置 |
|----|------|
| 目标 | 确定三段式机制各阶段的门槛数值（按品类分桶） |
| 数据 | 历史直播日志（近90天） |
| 分析维度 | 品类 × 语言 × 地区 |
| 关键指标 | 首分钟留存、评论/分钟、负反馈率 |
| 验收 | 各品类门槛建议值 + 置信区间 |

## Phase 1: 高潜主播识别

### MVP-1.1: 主播特征工程（开播前）

| 项 | 配置 |
|----|------|
| 目标 | 构建开播前可用的主播特征集 |
| 特征类别 | 账号历史、画像、可持续性线索 |
| 账号历史 | 短视频发布频率、完播率、互动率、粉丝结构 |
| 画像 | 内容品类、语言、地区、兴趣embedding |
| 可持续性 | 是否有规律产出、是否有稳定人设/主题 |
| 验收 | 特征覆盖率 + 特征重要性分析 |

### MVP-1.2: 主播特征工程（启动期）

| 项 | 配置 |
|----|------|
| 目标 | 构建开播后5-10分钟可用的启动期特征 |
| 特征类别 | 留存曲线、互动质量、内容健康度、转化漏斗 |
| 留存曲线 | 前30秒/1分钟/3分钟留存率、留存曲线形状 |
| 互动质量 | 评论/分钟、评论多样性、主播回复率 |
| 内容健康度 | 负反馈率、违规风险信号 |
| 转化漏斗 | 关注率、礼物率（非唯一） |
| 验收 | 特征工程完成 + 探索性分析 |

### MVP-1.3: 高潜主播预测模型

| 项 | 配置 |
|----|------|
| 目标 | 预测主播30/60天LTV或生存概率 |
| 标签 | 30天LTV（收入+粉丝增长价值+留存价值）或30天生存指标 |
| 模型 | GBDT + 生存分析 |
| 基线 | 纯规则分层 |
| 验收 | AUC>0.7, 可解释性报告 |

**Gate影响**: 
- AUC ≥ 0.7 → 上线分层探索预算（S/A层高预算）
- AUC < 0.6 → 需增加特征或调整标签

## Phase 2: 用户意图分层

### MVP-2.1: 用户意图分层定义

| 项 | 配置 |
|----|------|
| 目标 | 建立轻量可解释的用户分层 |
| 分层 | 付费高意图、社交互动型、内容消费型、新用户/冷启动 |
| 付费高意图 | 历史打赏、购买、强互动、深度停留 |
| 社交互动型 | 爱聊天、爱评论，但不一定付费 |
| 内容消费型 | 刷得快、偏短停留、少互动 |
| 新用户 | 无历史，需保守策略 |
| 验收 | 分层覆盖率 + 分层区分度分析 |

### MVP-2.2: 差异化排序权重实验

| 项 | 配置 |
|----|------|
| 目标 | 验证不同人群差异化排序权重的效果 |
| 实验设计 | Bandit或分桶A/B |
| 权重变量 | 礼物转化权重、互动质量权重、吸引力权重 |
| 评估指标 | 用户侧体验（留存、负反馈）+ 转化效率 |
| 验收 | Δ体验>5%且无显著负面影响 |

## Phase 3: 成长信号产品化（US特性）

### MVP-3.1: 成长信号产品设计

| 项 | 配置 |
|----|------|
| 目标 | 设计可见、可追踪、可达成的成长信号体系 |
| 成长仪表盘 | 本场新增粉丝、回访观众数、互动峰值、高光时刻 |
| 里程碑任务 | 首100粉/首50场有效直播/连续7天开播 |
| 社区认同 | 高质量评论置顶、优秀互动回放、本场最受欢迎话题 |
| 流量可解释 | "你通过了XX门槛→系统加码了曝光" |
| 验收 | 产品PRD + 原型 |

### MVP-3.2: 成长信号A/B实验

| 项 | 配置 |
|----|------|
| 目标 | 验证成长信号对主播开播留存的因果效应 |
| 实验设计 | 随机分流A/B |
| 对照组 | 无成长信号 |
| 实验组 | 完整成长信号体系 |
| 评估指标 | 7/30日开播留存、开播频次 |
| 验收 | Δ留存>10%且统计显著 |
